本期主要介绍一下Oracle中比较常用的方法和关键字

1、TRIM（String），RTRIM（String），LTRIM（String）

（1）语法：rtrim（String），trim（String），ltrim（String）。
（2）TRIM去除指定字符串string的左右空格，rtrim去右侧空格，ltrim去掉左侧空格。

2、TO_CHAR关键字：

（1）语法：to_char('20180102','yyyyMMdd');to_char(999,'99999'),to_char(125.8, '999D9')
（2）解释：将其他的数据类型转化为字节。
（3）使用：
    ①select to_char('20180102','yyyy-MM-dd') from dual；→"2018-01-02"
    ②select to_char(999,'99999') from dual；→转化后设置5个占位符 "  999"
    ③select to_char(125.8, '999D99') from dual；→转化后保留两位小数"125.80"

3、NVL关键字：

（1）语法：NVL(目标值,0)。
（2）解释:如果"目标值"是null的话，默认值为0。

4、split关键字（方法）：split方法创建和调用的方法如下：

创建：
create or replace function f_split(p_str varchar2,p_separator varchar2) return SPLIT_ARRAY is
     v_str varchar2(4000);
     v_split_array SPLIT_ARRAY := SPLIT_ARRAY();
     v_index binary_integer := 0;
   begin
     if p_str is null or p_separator is null then
       return v_split_array;
     end if;
     v_str := p_str;
     --先trim前后的p_separator
     --如果以p_separator开头则ltrim
     if instr(v_str,p_separator,1) = 1 then
       v_str := ltrim(v_str,p_separator);
     end if;
     ----如果以p_separator结尾则rtrim
     if instr(v_str,p_separator,-1) = (length(v_str) - length(p_separator) + 1) then
       v_str := rtrim(v_str,p_separator);
     end if;
     if length(v_str) = 0 then
       return v_split_array;
     end if;
     while instr(v_str,p_separator) != 0 loop
       --如果两个p_separator相邻则不加入v_split_array中
       if length(substr(v_str,1,instr(v_str,p_separator) - 1)) != 0 then
         v_index := v_index + 1;
         v_split_array.extend;
         v_split_array(v_index) := substr(v_str,1,instr(v_str,p_separator) - 1);
       end if;
       v_str := substr(v_str,instr(v_str,p_separator) + length(p_separator));
     end loop;
     if length(v_str) != 0 then
       v_split_array.extend;
       v_split_array(v_index + 1) := v_str;
     end if;
     return v_split_array;

  end f_split;
调用：
SELECT COLUMN_VALUE AS ID FROM TABLE(F_SPLIT(PRM_AAC020, ','))

5、MERGE INTO：

（1）语法：
merge into语法：
    merge into [target-table] a using [source-table sql] b on([conditional     expression] and [...]...)  
    when matched then  
        [update sql]  
    when not matched then  
        [insert sql]
（2）解释：判断Ｂ表和Ａ表是否满足on中条件，如果满足则用b表去更新a表，如果不满足，则将b表数据插入a表但是有很多可选项。
（3）实例：
           table1 a_merge 字段 id name year 
    table2 b_merge 字段 id aid name year city
    table3 c_merge 字段 id name city
    ①两张表操作：
    --MERGE INTO 主表；
    --USING()需要对比的，
    --on条件：如果 a_merge表的id=b_merge表的id，那么将 a_merge（主表）的name
    --字段更新为b_merge表对应的name，否则将b_merge的数据插入到a_merge（主表）中。
    MERGE INTO a_merge T1
          USING (SELECT * FROM b_merge) T2
    ON (T1.id = T2.aid)
    WHEN MATCHED THEN
         UPDATE SET T1.NAME = T2.NAME,T1.year=T2.YEAR
    WHEN NOT MATCHED THEN 
        INSERT (ID,NAME,YEAR) VALUES(T2.ID,T2.NAME,t2.year);
    ②单张表操作：
    --有些java框架中针对单张表的操作的时候会出现这样的操作
    --如果该条数据存在id 那么根据id做，
         --如果该条数据不存在id 那么做insert操作
    MERGE INTO LOGINPER T1
         USING (SELECT COUNT(1) AS ids FROM LOGINPER a WHERE a.pro001='1000112') T2
     ON (T2.IDS>0)
    WHEN MATCHED THEN
      UPDATE SET T1.PRO002 = '微信' WHERE T1.pro001='1000112'
    WHEN NOT MATCHED THEN 
      INSERT (T1.PRO001,T1.PRO002) VALUES(sys_guid(),'林夏');

6、WITH…AS…

（1）语法：基本语法格式→with 表别名 as sql
（2）解释：虚拟视图 可在复杂sql语句（或较大的sql）中使用可提高效率、提高代码的可视性和可续性。         
（3）实例：
    WITH 
       a AS (SELECT * FROM a_merge),
       b AS (SELECT * FROM b_merge),
        c AS (SELECT * FROM c_merge);
    select * from a,b where a.id = b.id;

7、 DECODE关键字

（1）语法：DECODE(运算结果,值1,值2,值3)
（2）解释：DECODE方法相当于if..else.. 也类似于三目运算符，
        如果“运算结果”= 值1的话→返回值2，否则如果不等于→返回值3。

8、DISTINCT关键字

（1）语法: Select DISTINCT STR001 from table1 where 条件1;
（2）解释：去重复值。
（3）DISTINCT函数：语法要求查询的结果一个或多个字段，如果后边跟两个（或以上）字段，查询结果是根据两个字段分别去重的。

9、身份证校验方法SUBSTR（截取）和MOD（取余）

设定值：ITEM_RYXX 18位身份证号
方法：mod(substr(ITEM_RYXX,15,1),2)
（1）substr（）：Oracle数据库字符串截取的方法。substr（“字符串”，起始点，结束点）
         注：起始点是从1开始不是从0开始，下边的例子：
         substr('201711211009365E65751B2426006AE053AC123233A6AF',1,4) 截取的值就是“2017”
（2）mod（）：Oracle数据库取余的方法。mod（目标值，除数）。

10、LENGTH关键字

（1）语法：length(“字符串”)
（2）解释：获取字符串的长度
（3）使用方法：Select length("ITEM_RYXX.Str002") from dual;

（1）语法：TO_DATE("1990-01-23", 'yyyy-mm-dd')。

13、TO_NUMBER关键字

（1）语法：to_number（'阿拉伯数字'）.
（2）解释：将字符串转化为数字.
（3）应用：
    ①SELECT to_number('1021') FROM dual; → 显示 1021；
    ②SELECT to_number('12454.8-','99999D9S') FROM dual; →显示 -12454.8；
    ③SELECT to_number('12,454.8-','99G999D9S') FROM dual;→ 显示 -12454.8；
    ④SELECT to_number
    ('12,454.8','99G999D9') FROM dual;→ 显示 12454.8；

17、BETWEEN…AND:在…之间

     语法：使用在where 之后 表示条件。
     解释：表示在“大于等于…小于”>= 并且 <。
     实例： SELECT aae036 FROM zw11 WHERE aae036 BETWEEN  to_date('2017/11/17', 'yyyy-mm-dd') and
          to_date('2017/12/6', 'yyyy-mm-dd') ;

18、TRUNC 关键字

     语法：在Oracle数据库中Trunc是截取函数用在日期类型上，截取到日期或时间 译：“剪枝”
     实例：

以下例子：日期以“2012-12-19 14:30:50”为例
（1）、 SELECT TRUNC(SYSDATE,’MI’) FROM DUAL;–默认截取到日，得到日期“2018/6/25”;
（2）、如果值为“2012-12-19 14:30:50”的日期变量 TRUNC(SYSDATE)得到日期为2012/12/19和TRUNC(SYSDATE,’DD’)一样;
（3）、如果TRUNC(SYSDATE,’YYYY’)，→ 2012/01/01;
（4）、如果TRUNC(SYSDATE,’MM’)，→ 2012/12/01;
（5）、如果TRUNC(SYSDATE,’DD’)，→ 2012/12/19;
（6）、如果TRUNC(SYSDATE,’HH’)，→ 2012-12-19 14:00:00；
（7）、如果TRUNC(SYSDATE,’MI’)，→ 2012-12-19 14:30:00;
（8）、如果TRUNC(SYSDATE,’SS’)，由于精度过高 不支持 报错;
（9）、获取当前日期所在周的起始时间
BETWEEN TRUNC(TO_DATE(TO_CHAR(SYSDATE, ‘YYYY-MM-DD’), ‘YYYY-MM-DD’), ‘IW’) AND
TRUNC(TO_DATE(TO_CHAR(SYSDATE, ‘YYYY-MM-DD’), ‘YYYY-MM-DD’), ‘IW’) + 6

19、round关键字：

     语法：round(number1,number2);
解释：number1 目标函数，number2保留的小数位数（写0或者不写默认为对整数位进行操作）；
     实例：
（1）select round(123.461) from dual;→ 123
（2）select round(123.467,1) from dual;→123.5
（3）select round(123.467,2) from dual;→123.47

20、MONTHS_BETWEEN 关键字：

     语法：MONTHS_BETWEEN(date1, date2) 
     解释：date1与date2相差多长时间，（date1时间稍晚一点 大的日期放在前边）
     实例：

（1）计算两个时间相差多长时间
SELECT MONTHS_BETWEEN(SYSDATE, to_date(‘2018-01-23’,’yyyy-MM-dd’)) FROM dual;→.4166
（2）计算两个时间相差年月
SELECT ROUND(MONTHS_BETWEEN(SYSDATE, to_date(‘2018-01-23’,’yyyy-MM-dd’)) / 12, 1);→.4（四个月）

21、AVG关键字

    语法： avg（字段名）获取平均值
    解释：Oracle中求统计平均值的方法；
    实例：

（1）获取某个字段的数据统计的平均值 ： SELECT AVG(a.agb012_1) FROM Gb01 a;

22、COALESCE关键字：

语法：COALESCE
实例：SELECT COALESCE(c1,c2,c3) AS ABC FROM Gc36; 
解释：依次参考各参数表达式，遇到非null值即停止并返回该值。如果所有的表达式都是空值，最终将   返回一个空值。

23、DEFAULT关键字：

场景：创建表字段设置默认值。
现场：当表中摸某个字段设置默认值时，数据在做insert操作的情况下会默认给定一个默认值。
insert时默认值失效的办法：针对当前的表单独创建一个视图view，针对这个视图保存的时候原表默认值属性属性失效。
单表视图是可以insert操作的

24、LEAD和LAG

场景：LEAD 需要获取当前数据下边几条数据 LAG获取当前数据上边第几条数据
情景：（1）、查询当前数据，获取当前数据下一条数据的部分信息
    SELECT B.*,
           LEAD(B.UNIVERSITYNAME, 1,  B.AUN001) OVER(ORDER BY B.UNIVERSITYNAME) AS N_AZD08A,
           LEAD(B.UNIVERSITYNUMVER, 1,  B.AUN002) OVER(ORDER BY B.UNIVERSITYNUMVER) AS N_AZD083,
           LEAD(B.UNIVERSITYSTR, 1,  B.AUN005) OVER(ORDER BY B.UNIVERSITYSTR) AS N_AZD081
      FROM UNIVERSITYINFO B 
     ORDER BY B.AUN001;
     （2）、查询当前数据，获取当前数据上一条数据的部分信息
     SELECT B.*,
           LAG(B.UNIVERSITYNAME, 1,  B.AUN001) OVER(ORDER BY B.UNIVERSITYNAME) AS N_AZD08A,
           LAG(B.UNIVERSITYNUMVER, 1,  B.AUN002) OVER(ORDER BY B.UNIVERSITYNUMVER) AS N_AZD083,
           LAG(B.UNIVERSITYSTR, 1,  B.AUN005) OVER(ORDER BY B.UNIVERSITYSTR) AS N_AZD081
      FROM UNIVERSITYINFO B 
     ORDER BY B.AUN001;
