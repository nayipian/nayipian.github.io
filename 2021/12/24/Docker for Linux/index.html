Docker for Liunx

发表于2021-12-24|更新于2022-02-27|文档教程
|阅读量:

前期准备

本机搭建起一台虚拟机搭载Linux启动，配置好虚拟机的环境 ip、网关以及DNS，确保Xshell访问和访问互联网

搭建和卸载

需要了解和注意的是docker分为两个版本一个是企业版一个是社区版，Docker要求CentOS系统内核版本要高于3.10。

（1）、进入到xshell 使用root用户登录linux
（2）、查看linux系统内核版本，命令：uname -r
（3）、更新yum包，将yum包更新到最新，命令：sudo yum update 等待下载更新完毕
（4）、安装需要的软件包，yum-util提供yum-config-manager功能，另外两个是devicemapper驱动以来的，命令：sudo yum install -y yum-utils device-mapper-persistent-data lvm2
（5）、设置yum源，命令：sudo yum-config-manager –add-repo https://download.docker.com/linux/centos/docker-ce.repo
（6）、查看docker文件源，命令:yum docker-ce –showduplicates | sort -r
可以查看到各个版本的docker
（7）安装docker，命令:
方式一，安装最新版本：sudo yum install docker-ce
方式二，安装指定版本：sudo yum inatall docker-ce-17.12.1.ce
（8）、查看版本：命令 docker version 或者 docker -v
（9）、启动docker，命令：sudo systemctrl start docker ，可以选择添加启动命令，sudo systemctrl enable docker。启动完成后可以使用docker version 查看一下，可以发现此时多了一个服务端。
（10）、停止docker，命令：sudo ssystemctrl stop docker
（11）、配置容器镜像加速器提高效率。大多数都推荐使用阿里云的容器镜像加速器，在容器镜像服务中，找到镜像中心，找到镜像加速器，里边有一个操作文档一栏。选择centos tab页，逐句执行命令。
（12）、下载hello-world镜像，命令：docker pull hello-world 。执行命令 sudo docker image ls ，可以查看到hello-world。通过使用 sudo docker run fce 可以看到“Hello from Docker！” 说明安装成功。
（13）、卸载docker命令:sudo yum remove docker docker-common docker-selinux docker-engine

Docker 镜像

语法：

  • 查询镜像：docker images [镜像名:tag]
  • 查询所有镜像：docker images -a
  • 拉取镜像：docker pull [镜像名:tag]
  • 推出镜像：docker push [镜像名:tag]
  • 删除镜像：docker rmi [镜像名:tag，或者镜像id]

Docker 容器

语法：

  • docker ps 查看所有正在运行的容器
  • docker ps - a 查看所有的容器
  • docker ps - f 查看所有的停止容器
  • docker ps - l 查看最后运行的容器
  • docker rm [容器名或者容器id] 删除容器
  • 运行容器：docker run [-di后台启动] --name [启动的容器名] -p [端口映射] [镜像名] ,例如： 在run的过程中 如果本地有镜像就会使用本地的，如果本地没有镜像就需要到远程仓库中获取完事之后再启动
  • docker stop [容器名或者启动容器id] 停止容器
  • docker start [容器名或者启动容器id] 启动容器
  • 进入容器：docker exec -it [容器启动id] /bin/bash, **/bin/bash**以命令行的方式进入到容器，以centos为例
  • 退出容器：exit
  • 拷贝资源 docker cp [宿主机上单资源地址] [容器名称]:[对应容器的地址] 部署环境的时候可以将jdk、工程包、tomcat等外部文件通过这样的方式宿主机传入到容器

Dockerfile的基础语法

1.1 FROM

语法：FROM <image>:<tag>
指明构建的新的镜像是来自于哪个基础镜像。如果没有tag，那么默认值为 latest

1.2 MAINTAINER(deprecated)

语法：MAINTAINER<name>
指明镜像的维护者及其联系方式（一般是邮箱）。官方说明已经过时了，推荐使用LABEL。

1.3 LABEL

语法：LABEL <key>=<value> <key>=<value> <key>=<value> ...
功能是为镜像指定的标签。也可以使用LABEL来指定镜像的作者。

1.4 RUN

语法：RUN<command>
构建镜像时运行的Shell命令，比如构建新镜像中我们想在 /usr/local 目录下创建一个java目录

1.5 ADD

语法：ADD <src>....<desc>
拷贝文件或目录到镜像中，src可以是一个本地文件或者是本地压缩文件，压缩文件会自动解压，还可以是一个url，如果把src写成一个url，那么ADD就类似一个wget命令，然后自动下载和解压。

1.6 COPY

语法：COPY <src> ...<dest>
拷贝文件或者目录到镜像中，同ADD，区别在于COPY就是单纯的拷贝不会支持自动下载和解压

1.7 EXPOSE

语法：EXPOSE <post> [<post>/<protocol>...]
暴露容器运行时候的监听端口给外部，可以指定端口是监听TCP还是UDP，如果未指定协议，则默认是TCP.

如果想要容器和宿主机的端口有映射关系，必须在容器启动的时候加上 -P参数

1.8 ENV

语法：ENV <key><value> 添加单个，ENV <key>=<value> ... 添加多个。

1.9 CMD

语法：

  • CMD ["executable","param1","param2"],比如：CMD ["/usr/local/tomcat/bin/catalina.sh","run"]
  • CMD ["param1","param2"],比如：CMD ["echo","$JAVA_HOME"]
  • CMD command param1 param2 ，比如：比如：CMD echo $JAVA_HOME
    启动容器时执行的Shell命令。在Dockerfile中只能有一条CMD命令，如果设置多条CMD命令，只有最后一个CMD命令生效。使用的时候需要注意的是CMD中间有一个空格在+[] 如果创建容器的时候指定了命令，则CMD命令会被代替。假设镜像centos:7 ，创建容器时候的命令是 docker run -it –name centos7 centos:7 echo “helloworld” 或者 docker run -it –name centos7 centos:7 /bin/bash,就不会输出 $JAVA_HOME 的环境变量信息了，因为CMD命令会被 echo “helloworld”、/bin/bash 覆盖了。

1.10 ENTRYPOINT

语法：

  • ENTRYPOINT ["executable","param1","param2"],比如 ： ENTRYPOINT["/usr/local/tomcat/bin/catalina.sh","run"]

  • CMD command param1 param2 ,比如：ENTRYPOINT echo $JAVA_HOME
    容器启动时执行的Shell命令，与CMD类似，但是ENTRYPOINT不会被 docker run 命令行执行的参数所覆盖。在DockerFile中只能有一条ENTRYPOINT命令。如果设置了多条的ENTRYPOINT，只有最后一个ENTRYPOINT会生效。

  • 如果在dockerfile中同时有 CMD 和ENTRYPOINT，并且CMD指令不是一个完成的可执行命令，那么CMD指定的内容将会作为ENTRYPOINT的参数

  • 如果在dockerfile中同时有 CMD 和ENTRYPOINT，并且CMD是一个完整的指令，那么两个会互相覆盖，谁在最后谁生效。

1.11 WORKDIR

语法：WORKDIR /path/to/workdir
为 RUN、CMD、ENTRYPOINT以及COPY和AND设置工作目录，

1.12 VOLUME

语法：volume ["/var/bin/mysql"]

指定容器挂载点到宿主机自动生效的目录或其他容器。一般的使用场景为需要持久化存储数据时。
容器的 /var/bin/mysql 目录会在运行时自动挂载为匿名卷，匿名卷在宿主机的 /var/bin/docker/valumes目录下。

  • 一般情况下不会在dockerfile中使用，更常见的还是在docker run的时候通过 -v 指定数据卷

1.13 使用dockerfile文件

通过dockerfile文件构建一个容器，首先需要使用vim创建一个Dockerfile文件，文件名字就是 dockerfile ，使用Dockerfile的基础语法构建一个简单的dockerfile文件，如下

命令：docker build -f [指定文件（dockerfile地址）] -t [镜像构建完成后的名字] [构建时所需的资源]

构建时所需的资源指的是构建容器时需要的jar包，.gz压缩包等依赖文件，可以对应Dockerfile ADD关键字对应的资源，如果不指定会报错，指定文件方式如下：

  • 资源文件的绝对路径 例如：/usr/local/docker/dockerfile
  • 资源文件的当前路径,使用“.” 来标识，例如：
    docker build -f /usr/local/docker/dockerfile/Dockerfile -t mycentos:7 .

Docker的网络模式

docker的运行是以容器的方式进行的，以liunx为例，一台宿主机是linux上边可能运行很多的docker容器，容器和容器之间，容器和宿主机之间都需要进行数据的传输和通信，这样情况在集群部署最为常见。
容器在运行的时候都会有一个ip地址，通过使用命令： docker inspect [容器名或容器id] ，可以查看到当前运行容器的ip

容器ip是随机变化的不是固定的，随着容器的启动和管理，每一次ip都是不一样的。

刚刚创建的docker容器有三个网络模式（bridge、host、none） 通过命令 docker network ls 查看

1、桥接模式（bridge）

为每一个容器分配一个自己的ip地址，并且为创建一个docker0的虚拟网桥，通过docker0的虚拟网桥进行连接。当安装docker引擎之后随之会创建docker0的虚拟网卡，容器默认也是桥接模式。

在该模式中docker守护进程，创建一个以太网docker0，docker0中有一个evth，创建容器之后会有一个docker-eth0，通过宿主机器上的docker0-evth进行通信，容器中的docker-eth0与宿主机上的docker0-evth是一段对等设备，对等设备一个在docker0中一个在容器中实现通信。

2、host模式

容器不在自己创建IP地址和网关等，所有的网络全部基于宿主机。
命令：docker run -it –name bbox –network host daocloud.io/library/busybox:1.26-glibc，启动的时候指定网络模式，启动完成后ipaddr查看，所有的网络都会与宿主机保持一致。

3、none模式

容器具备独立的networknamespace，初次之外没有任何设置，网络模式基本上都是空的需要手动配置。

4、container模式

新创建的容器不会有自己的ip和网卡，而是基于一个指定的容器共享ip等网络配置。**–network container：正在运行的容器名称或ip**
命令：docker run -it –name bbox01 –network container：bbox daocloud.io/library/busybox:1.26-glibc，ipaddr 查看之后可以看到网络信息与bbox容器一致

5、link模式

通过命令docker run –link实现容器之间的通信，究其底层是修改了host，vim /etc/hosts,不过官网不推荐这种方式。
