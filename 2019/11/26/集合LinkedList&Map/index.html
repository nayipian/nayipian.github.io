ArrayList和LinkedList的比较

ArrayList

ArrayList是在集合体系中最开始接触的集合，也是在实际开发中出镜频率比较高的集合。基本模式

ArrayList底层是现实是数组，既然是数组，那么很多情况ArrayList与数组关系很密切。对于数组来说其特点之一就是在初始化的时候需要指定其长度，作为对数组的升级版，ArrayList不需要在初始化在定义长度，而是实现了动态数组长度的变化。
对此提出疑问ArrayList是如何实现动态数组长度的变化？
基本原理是：ArrayList当它在调用add()方法的时候，首先做的是判断当前数组是否还可以放下要添加的数据，如果可以放下，就会将该数组元素放在上一个数据的后边；如果放不下，ArrayList将会新开辟一个数组，同时将原数组中的数据copy到新数组中同时在将新的数据放在新copy的数据后边。

ArrayList的特点：查询快，增删慢，消耗内存。
（1）查询快：因为数组的查询是根据“索引”角标进行的检索，因此查询的速度比较快。
（2）增删慢：主要的原因也是因为数组和Arraylist的特性造成的，首先但add值的时候需要计算出指定位置，并且将指定位置的值一次向后移动，当remove值的时候，删除之后后边的值需要往前移动一位，当集合内的数据，也就是说size的值过大的时候就会是效率变得非常的底。
（3）消耗内存：当针对一个ArrayList频繁的进行add操作的时候，底层数组的实现方式是将原数组copy再在后边添加一个新的元素，当创建数据处理完成之后原数组就被是为存储的垃圾会被jvm回收，并且每次复制新数组都会在原数组的基础之上扩容都1.5倍，扩容的部分如果数据放不满就会被空限制，因此频繁的对Arraylist进行add操作系统内存的消耗是比较大的。

LinkedList

1、LinkedList集合是链表是集合，主要分为：单链表、双链表和循环链表。主要模式

链表的模式可以形象的想象成为一个链条将数据有序的连接起来。这里的每个元素都称之为node节点。整合起来链表就是使用链条模式向node连接起来。

2、node节点的和链表结构

以上程序中的add方法就是在不断创建node节点，每个node节点都对应有一个内存地址，在一个node节点中会存放着两个元素，一个是value值（11,12,13….），另个则是Node next（内存地址值）。node next在第一次创建的时候是null，当下一次创建node时，新的node会把自己的内存地址给上一个node next，这个就形成了node和node之间的关系。
这里需要了解的是由于node每一次都是在内存中new出来的，因此node的元素不是连续的，所以LinkedList元素也是不连续的。

当创建一个LinkedList对象的时候，LinkedList在堆内存中也会存在一个内存地址，在linkedList内部也会存放着两个成员，一个是size，记录着node元素的个数，另一个是node frist，存放着node链的第一个内存地址值，也就是说Linkedlist也指向了第一个node节点，这样一来一个完整的链条结构就完成了。

3、LinkedList的优势和不足。
优势：优化内存空间。
（1）从add的角度：从它的底层模式不难看出LinkedList是每一次创建的时候占据一块内存空间，将自己的内存地址赋给上一个node 的node next，整体不没有多余的空余的浪费。
（2）从remove角度：当删除集合中的某一个node时，被删除的node连带内存地址一起被jvm回收，上一个node节点的node next会自动指向被删除node的下一个node节点上。这样会很容易的释放了空间。
不足：检索慢
（1）、检索慢：因为LinkedList底层的实现是链表模式，因此没有角标索引的感念因此检索的速度远远比不上ArrayList；
检索流程是首先需要找到LinkedList中的next frist对应的内存地址 在拿到node next 实际是一个循环遍历取值。

4、LinkedList的特点：
不连续：上边已经提及到了不连续的原因，这是由实现模式决定的，因为每一次new node时内存地址不连续，因此LinkedList里边的内容不连续。

LinkedList单项列表和双向列表

在实现的方式和对外提供的方法上看LinkedList和ArrayList存在很大的相同点，这样的做的主要目的可能处于集合体系的完整性。
单项列表的主要含义就是从LinkedList出发放最后一个node方向；
双向列表就是正好相反，此时需要明确一点LinkedList中会多了一个成员就是node last 用来记录后一个node的内存地址。同时每个node中会增加一个记录前一个node内存地址的参数pre。这样双向列表整体模式就是获得了。

1、异常
当LinkedList集合为空，也就是说集合中没有数据无论是get还是remove都会抛出异常：没有元素异常

jdk1.6以后 LinkedList退出了一种新的方式：获取peekFirst();/peekLast();移除pollFirst()/pollLast;方法，在集合中没有数据的情况下不抛出异常而返回空值null，一定程度可以替代之前的方法。

一、jdk1.7 HashMap

1、首先需要知道的是HashMap的在Jdk1.7和jdk1.8存在不同
从底层实现上看：
jdk1.7底层实现：数组+链表
jdk1.8底层实现：数组+链表+红黑树

2、使用：
先创建一个HashMap

先通过key计算一个hash值(int hash = hash(key))在通过一个hash值调用indexFor（ int i = indexFor(hash, table.length)）方法计算出来一个i。i的计算结果与table有关，这个table就是hashmap的数组（数组有容量），这个i其实就是hashmap数组的下标，table的类型就是Entry，在源码Entry中包含了以下内容：key value next hash

在hashmap中当有一个元素，需要放进数组的时候需要经历以下步骤：
(1)、元素中包含key和value，先需要对key进行hash运算得到一个hash码(int hash = hash(key))
(2)、获取数组下标。需要对hash码取模求余运算（int i =hash % table.length()）
(3)、如果当前获得的运算结果也就是下标与之前的下标重复，那么之前的元素和当前元素会通过链表进行关联，当前元素出于效率考虑将会放在原来元素的头部节点，此时next方法就会将地址指向头节点并把当前节点作为当前数据的头节点。

二、jdk1.8 HashMap

由于jdk1.7中的hashmap底层实现是数组+链表，考虑到效率上原因链表查询效率低，为了完善jdk1.8中引入新的实现方法链表+数组+红黑树是实现模式。

当链表的长度到达一定数量（通常认为>8），并且数组容量超过64（jdk1.8 引入属性 static final int MIN_TREEIFY_CAPACITY = 64; 启用红黑树的最小元素个数）的时候，有可能将链表转化为红黑树，否则实现模式仍然是链表，采取的方案是扩容。红黑树的效率相比数组依然不是很高，但是比起链表插入和查询的效率是折中的。
参照jdk1.8的源码可以发现，引入树形结构以外其他的实现模式基本相同，但有一些属性值出修改
